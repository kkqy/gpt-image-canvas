# References

This directory is reserved for long-form LLM reference files that are useful to future agents.

Current scope: scaffold only. Do not add broad references such as Nixpacks, `uv`, or unrelated deployment guides until the repository actually uses or needs them.

When adding a reference:

- Prefer focused, project-relevant material.
- Name files with a clear `*-llms.txt` suffix when they are meant to be loaded by agents.
- Link the reference from the relevant top-level guide.
- Keep secrets, local paths, run logs, and generated assets out of reference files.
