# Active Execution Plans

Put in-progress execution plans here when a task is too large for a single direct implementation pass.

Each active plan should include:

- Goal and user value.
- Affected areas: `apps/api`, `apps/web`, `packages/shared`, docs, Docker, or data.
- Implementation steps.
- Interfaces and schema changes.
- Verification commands.
- Browser verification requirements, if UI changes.
- Security and reliability notes.

Move completed plans to `docs/exec-plans/completed/` or replace them with a durable product spec when the work becomes product policy.
